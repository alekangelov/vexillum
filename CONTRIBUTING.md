## Tech stack
- Backend: Rust with Axum
- Database: PostgreSQL
- Frontend: Solid with Tailwind CSS
- Infrastructure: Docker, Docker Compose
- CI/CD: GitHub Actions
- Testing: Jest, Playwright
- Linting and Formatting: ESLint, Prettier, Rustfmt, Clippy
- Monitoring: Prometheus, Grafana
- Authentication: JWT, OAuth2

## Contributing
We welcome contributions from the community! Whether you're fixing bugs, adding new features, or improving documentation, your help is appreciated. Here are some guidelines to get you started:
1. Fork the repository and create your branch from `main`.
2. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
3. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations, and container parameters.
4. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent. We use [Semantic Versioning](https://semver.org/).

Please refer to the [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for more information on our code of conduct.
